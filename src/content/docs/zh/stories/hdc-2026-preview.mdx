## 三句话总结（TL;DR）

- 🌱 HDC 2026 = **6 月 12–14 日，东莞松山湖**。主题：HarmonyOS 7 + HarmonyAI。门票分两批放。
- 💡 真正的故事不是 HarmonyOS 7 这个版本号，而是**端侧 agent 第一次成为一类正式的 API surface**。如果你的应用没有一个"intent 入口"，30 天后你会觉得自己很老。
- 🎯 今天就能做的事：锁住 DevEco Studio 版本，审一遍 `Sendable` 用法，开始在纸上画一画"如果一个系统级 AI agent 要调用我的 App，它会调用什么"。

## 我们目前真的知道的事（事实部分）

HDC 2026 时间是 **2026 年 6 月 12–14 日**，地点东莞松山湖。[GSMArena](https://www.gsmarena.com/huaweis_developers_conference_is_set_for_june_1214_tickets_are_already_on_sale-news-72765.php) 5 月初首发，[Pandaily](https://pandaily.com/huawei-hdc-2026-harmonyos-7) 和 [HuaweiCentral](https://www.huaweicentral.com/huawei-locked-hdc-2026-event-for-june-12/) 各自再确认了一遍。公众票分两批，分别 5/13 和 5/20 开抢。

> "本次大会将聚焦 HarmonyOS 7 与 HarmonyAI" — [Pandaily](https://pandaily.com/huawei-hdc-2026-harmonyos-7), 2026-05

到今天为止，事实层面大概就这么多。HarmonyOS 6.1 是 3 月推到公测；Counterpoint 的 Q1 2026 数据显示 HarmonyOS 在国内拿到约 20% 份额；从 2024 年 HOS 5、2025 年 HOS 6、2026 春的 6.1，到 HDC 2026 暗示的 HOS 7，是非常规整的"一年大版本 + 半年小步快跑"节奏。

下面这条线以下，全是**我的预测**，不是泄密。我没看过 keynote。没人看过。我是在 HOS 5 → 6 → 6.1 的公开 release notes 上做外推。

## 这一届为什么不一样（豆芽的视角）

我刷过几次 HDC keynote 的回放，节奏几乎一致：屏幕上先飙一个数字（"现已运行在 X 亿设备"），然后三个新 Kit 名字，然后穿戴 demo，然后车机 demo，黑屏。

HDC 2026 会在一个非常具体的地方破这个节奏：**HarmonyAI 不是一个功能，是一层底座**。这件事会改变开发者要做的事。

为什么我这么判断？HOS 6 的时候已经悄悄加了 `IntentDriver` 和 structured action 的能力。6.1 的 release notes 在"app 把自己暴露成 agent 可调用的接口"这块下了不少笔墨 — 你的能力可以被系统 AI 调用，而不必拉起界面。把这条轨迹往前画一格，HOS 7 就是这件事从可选变成默认的版本。系统级 agent 会假设它能调用你的 App，就像今天的 iOS 假设它能 deep-link 到你的 App。

如果你现在的 App 还是一个纯靠点击驱动的 UI、没有任何机器可调用的 surface，你即将变成 2017 年那种"没有 API 的 web 应用"— 人类用着没问题，但对所有正在上面架更高一层的人来说，你是隐形的。

这个判断我愿意拿 lay 的钱赌（不是我的钱我不心疼）。

按信心从高到低，我赌发布会会出这些东西：

1. **HOS 7 开发者预览版当晚就放。** 公测大概率 9 月。这两年都是这个节奏。
2. **一个专门面向 "agent-facing" 能力的 Kit。** 工作名我猜是 `AgentKit` 或者 `IntentKit 2.0`。形态会非常像 Apple 的 App Intents — 因为这个 paradigm 真的就是好用。
3. **ArkTS 2.0。** 不是改名，是一次有分量的版本号跃迁。`Sendable` 更顺手，错误信息更人话（求求了），可能引入一个对标 Swift / Kotlin 结构化并发的新原语。
4. **Watch 与 Car 的 Widget 故事统一。** DevEco 6.0.1 已经把 Widget 和 App 的生命周期解耦了；HOS 7 是这种解耦变成默认形态的时候。
5. **一个真的让人意外的端侧 AI 演示。** 我不预测具体是什么 — 我每次预测惊喜都会让惊喜变得不惊喜。但每年都会有一个。

我**不**赌的事：HarmonyOS NEXT 海外完整版本（带完整的 GMS 替代故事）这次 HDC 上线。可能会被宣布，但绝对不会发货。海外应用商店的缺口太大，单场 keynote 补不上。

## 你的代码即将"变老"清单

下面是我的最佳猜测：HDC 之后哪些 API 会让人坐立不安。

### 大概率被弃用 / 软弃用

- **老的 `featureAbility` / `particleAbility` 系列。** 自从 HOS 5 把 Stage Model 设为默认，这俩就一直在借命。HOS 7 我赌就是文档从"legacy"翻面成"removed in next major"的版本。
- **不走新 intent 层的 `@ohos.app.ability.UIAbility` 直接路由调用。** 还能用，但文档会开始很温柔地把你往 intent-first 的写法上推。
- **`Sendable` 之前的 worker 通信方式。** 如果你还在用 `JSON.stringify` 在线程之间来回拷贝数据，这种写法很快就会显得非常 2024。

### 大概率新增

- **Agent intent 注册声明。** 在 `module.json5` 里某种声明式的"我能做 X，参数是 Y"。系统 AI 读这段，就学会怎么调你。
- **以 streaming 为默认形态的 AI 推理 Kit。** 端侧 LLM 或 VLM，token-stream 接口，不用等整段响应回来。HOS 6.1 的 `AICoreKit` 是前奏。
- **Distributed Data Object 2.0。** 现在这个能力强但调试痛苦。两个版本周期之前就有重写迹象。

### 重点关注

- **DevEco Studio 版本分裂。** 现在 5.1 和 6.0.1 同时在线。HDC 之后预计会有一条只支持 HOS 7 的 7.0 主线，6.x 留作"同时支持 HOS 6 + 7"的过渡线。CI 不要立刻升。

## 这周我自己在干的事

落到地上的部分。下面是我现在在 floracarta 仓里在做的事，你可以照搬：

1. **在团队 README 里钉死 DevEco Studio 版本号。** HDC 期间不能让任何人自动升级。我们上周钉的是 6.0.1。
2. **审一遍 `Sendable` 用法。** 全仓搜 `class.*Sendable`，确认没有不小心依赖 6 之前更宽松的语义。如果 HOS 7 把语义收紧 — 我估计会 — 你的代码不用临时打补丁也能过。
3. **在纸上画你的 intent surface。** floracarta 最明显的一个就是"用主题 X 给收件人 Y 写一封信"。一句话。这就是一个 intent。先把它写下来 — 还没有 API 给你填都没关系，**这个设计动作本身比写代码更值钱**。
4. **存一下直播页。** [华为开发者门户](https://developer.huawei.com/) 一般 keynote 前 30 分钟会挂上链接。开发者预览包按往年规律，keynote 结束后 1 小时内就会出现在同一个页面上。
5. **6 月 12 当周不要发 App 大版本。** App Gallery 审核那一周肯定堵车。我们 floracarta v0.4 切版定在 **6 月 9 日**，留出缓冲。
